## 1. 개요

- 배열의 ID는 명령문에 사용된 **`냥`의 개수(N)** 으로 결정됩니다.
- 변수와 배열의 ID가 같아도 서로 **별도로 취급**됩니다.
- 모든 배열의 인덱스는 **0부터** 시작합니다.
- 배열의 모든 요소는 **0으로 초기화**됩니다.

> `냥냥`은 변수2이면서 동시에 배열2를 가리킬 수 있습니다. 충돌 없이 독립적으로 동작합니다.

## 2. 배열 생성 (ARRAY_DECL)

```
냥N!<길이>야옹
```

- `냥 N개` → 배열 ID(번호)
- `!<정수형>` → 정수 리터럴을 길이로 사용
- `!<변수형>` → 해당 변수의 값을 길이로 사용

**예시**

```
냥!.....야옹       # 배열1 생성, 길이 5
냥냥!...야옹       # 배열2 생성, 길이 3
냥냥냥!냥야옹      # 배열3 생성, 길이 = 변수1의 값
```

## 3. 배열 값 쓰기 (ARRAY_WRITE)

```
냥N!<인덱스>!<값>야옹
```

- 첫 번째 `!` 뒤: 쓸 위치의 **인덱스**
- 두 번째 `!` 뒤: 배열에 저장할 **값**
- 인덱스와 값 각각 정수형/변수형 자유롭게 조합 가능

**조합표**

| 인덱스 | 값 | 예시 |
|--------|----|------|
| 정수형 | 정수형 | `냥!.....!......야옹` |
| 정수형 | 변수형 | `냥!.!냥냥야옹` |
| 변수형 | 정수형 | `냥!냥냥!...야옹` |
| 변수형 | 변수형 | `냥!냥냥!냥냥냥야옹` |

**예시**

```
냥!.....!......야옹        # 배열1[5] = 6
냥냥!.,!....야옹           # 배열2[0] = 4
냥냥냥!냥!.,야옹           # 배열3[변수1] = 0
냥냥냥냥!냥냥!냥냥냥야옹   # 배열4[변수2] = 변수3
```

## 4. 배열 값 읽기 (ARRAY_READ)

```
냥N!?<인덱스>!?<목적지>야옹
```

- `!?` 뒤 첫 번째: 읽을 위치의 **인덱스**
- `!?` 뒤 두 번째:
  - `~` → 읽은 값을 **스택에 push**
  - `냥M` → 읽은 값을 **변수M에 대입**

**조합표**

| 인덱스 | 목적지 | 예시 |
|--------|--------|------|
| 정수형 | 스택 | `냥!?.....!?~야옹` |
| 정수형 | 변수형 | `냥냥!?.,!?냥냥야옹` |
| 변수형 | 스택 | `냥냥냥!?냥!?~야옹` |
| 변수형 | 변수형 | `냥냥냥냥!?냥냥!?냥냥냥야옹` |

**예시**

```
냥!?.....!?~야옹             # 배열1[5] 값을 스택에 push
냥냥!?.,!?냥냥야옹           # 배열2[0] 값을 변수2에 대입
냥냥냥!?냥!?~야옹            # 배열3[변수1] 값을 스택에 push
냥냥냥냥!?냥냥!?냥냥냥야옹   # 배열4[변수2] 값을 변수3에 대입
```

---

## 5. 배열 입력 (ARRAY_INPUT)

```
냥N!<인덱스>??야옹
```

사용자로부터 정수를 입력받아 배열N의 특정 인덱스에 저장합니다.  
인덱스는 정수형 또는 변수형 모두 사용할 수 있습니다.

```
냥!.??야옹       # 배열1[1] ← 사용자 입력
냥!냥냥??야옹    # 배열1[변수2] ← 사용자 입력
```

## 6. 종합 예제

### 배열에 값 저장 후 읽기

```
# 배열1 생성 (길이 3)
냥!...야옹

# 배열1에 값 저장
냥!.,!...야옹     # 배열1[0] = 3
냥!.!.....야옹    # 배열1[1] = 5
냥!..!.......야옹 # 배열1[2] = 7

# 배열1[1] 값을 변수1에 읽어서 출력
냥!?.!?냥야옹         # 배열1[1] → 변수1
냥!?야옹              # 변수1 출력 → "5\n"
```
