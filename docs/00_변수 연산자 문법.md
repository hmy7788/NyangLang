# NyangLang 기본 문법
본 문서는 NyangLang의 `기본 문법`을 정의한다.
1. 기본 개념
2. 변수
3. 연산




## 1. 기본 개념
### 1.1 프로그램 구조
- `한 줄`은 `하나의 명령문`이다.
- 모든 계산에 사용되는 숫자와 연산 결과는 `스택(stack)` 자료구조를 통해 관리된다.
- 모든 의미는 해당 줄에 포함된 세 가지 요소로 결정된다.
  1. 냥의 개수 (ID)
  2. 냐의 존재
  3. 줄 끝 문자 (`.`, `~`, `!`, `?`)

그 외의 모든 문자(공백, 한글, 숫자, 영어, 기호)는 주석으로 간주하여 무시한다.

### 1.2 ID 규칙
- 한 줄에 등장하는 `냥`의 개수를 N이라 할 때, 해당 줄의 ID는 N이다.
- ID는 다음과 같이 사용된다:
  - 변수 번호
  - 함수 번호
  - 배열 번호
  - 조건문/반복문의 제어 변수




## 2. 정수형
정수: 온점, 반점의 갯수로 나타냅니다. 온점의 갯수만큼 1을 더하며, 반점의 갯수만큼 1을 뺍니다.
예시:
```
...   # 3
,,,   # -3
.,,   # -1
```




## 3. 변수(Variable)와 상수
### 3.1 변수 선언
형식:
```
1. <냥 N개><.과 , 조합>
```
의미:
- `N은 냥 개수`이며, `변수 번호`를 나타낸다.
- `.과 , 조합`은 변수의 `초기값`을 나타낸다. <br>
예시:
```
냥             # 변수1 = 0
냥냥....       # 변수2 = 4
냥냥냥.....    # 변수3 = 5
냥냥냥냥,,,    # 변수4 = -3
냥냥냥냥냥..,  # 변수5 = 1
```




### 3.2 변수 값의 사용과 대입
형식:
```
1. <. 과 , 조합>~
2. <냥 N개>~
```
의미:
1. `<. 과 , 조합>~` : 정수형을 스택에 push 한다.
2. `<냥 N개>~` : 문맥에 따라 두 가지로 해석된다. (문맥 기반 해석)
    1. `바로 이전 명령어`가 `연산 명령어가 아닌 경우`  
       - 변수 N의 값을 스택에 push 한다.
    2. `바로 이전 명령어`가 `연산 명령어인 경우`  
       - 스택의 top 값을 1개 pop 하여 변수 N에 대입한다.
       - (참고) 연산 명령어는 스택에서 2개를 pop하고, 결과를 1개 push 한다.

추가 규칙(구현 기준):
- `연산 명령어`가 실행되면 “직전이 연산이었다” 상태가 1회 활성화된다.
- 이 상태는 다음에 등장하는 `<냥 N개>~`에서만 소비되며, 대입이 수행된 뒤 비활성화된다.

예시:
```
[1번 케이스]
...~        # 스택에 1을 push
.,.,~       # 스택에 0을 push

[2-1번 케이스]
냥..        # 변수1 = 2
냥냥...     # 변수2 = 3
냥~         # 스택에 2 push & last_result 갱신
냥냥~       # 스택에 3 push & last_result 갱신

[2-2번 케이스]
냥..        # 변수1 = 2
냥냥...     # 변수2 = 3
냥냥냥      # 변수3 = 0
냥~         # 스택에 2 push & last_result 갱신
냥냥~       # 스택에 3 push & last_result 갱신
냐냐~       # 덧셈 연산: 스택에서 2개 pop(2,3) → 5를 push, last_result=5
냥냥냥~     # 직전이 연산 명령어이므로: 스택 top(5) 1개 pop → 변수3 = 5
```




## 4. 연산(Operation)
`모든 연산은 스택 기반으로 수행`된다. <br>
절차:
1. 스택에서 두 개의 값을 pop
2. 지정된 연산을 수행
3. 연산 결과를 스택에 push
4. last_result를 해당 결과로 갱신

### 4.1 연산 규칙
형식:
```
<냐 N개>~
```
|냐 개수|연산|
|---|---|
|2|덧셈|
|3|뺄셈|
|4|곱셈|
|5|나눗셈|
