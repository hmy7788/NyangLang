## 1. 기본 개념
### 1.1 프로그램 구조
- `한 줄`은 `하나의 명령문`이다.
- 모든 계산에 사용되는 숫자와 연산 결과는 `스택(stack)` 자료구조를 통해 관리된다.
- 모든 의미는 해당 줄에 포함된 세 가지 요소로 결정된다.
  1. 냥의 개수 (ID)
  2. 냐 또는 냐옹의 존재
  3. 줄 끝 문자 (`.`, `~`, `!`, `?`)

그 외의 모든 문자(공백, 한글, 숫자, 영어, 기호)는 주석으로 간주하여 무시한다.

### 1.2 ID 규칙
- 한 줄에 등장하는 `냥`의 개수를 N이라 할 때, 해당 줄의 ID는 N이다.
- ID는 다음과 같이 사용된다:
  - 변수 번호
  - 함수 번호
  - 배열 번호
  - 조건문/반복문의 제어 변수



## 2. 변수(Variable)와 상수
### 2.1 변수 선언
형식:
```
<냥 N개><. M개>   (N,M ≥ 1)
```
의미:
- N은 냥 개수이며, 변수 번호를 나타낸다.
- M = 점 개수이며, 변수의 초기값을 M-1로 나타낸다.
예시:
```
냥...          # 변수1 = 2
냥냥....       # 변수2 = 3
냥냥냥.....    # 변수3 = 4
```

### 2.2 상수
형식:
```
1. <. N개>~
2. ~<. N개> 
```
1. <. N개>~: `.`이 N개이고, `뒤`에 `~`가 온 줄은 상수 +N을 스택에 push & last_result를 갱신 
2. ~<. N개>: `.`이 N개이고, `앞`에 `~`가 온 줄은 상수 -N을 스택에 push & last_result를 갱신 
예시:
```
~       # 상수 0를 스택에 push & last_result 갱신
.....~  # 상수 5를 스택에 push & last_result 갱신
~.....  # 상수 -5를 스택에 push & last_result 갱신
```

### 2.3 변수 값의 사용과 대입
형식:
```
<냥 N개>~
```
- 해당 문법은 문맥에 따라 두 가지로 해석 (문맥 기반 해석)
  1. `바로 이전 명령어`가 `연산 명령어가 아닌 경우`, 변수 N의 값을 `스택에 push`
  2. `바로 이전 명령어`가 `연산 명령어인 경우`, 변수 N에 `last_result 값을 저장`, 그 다음 `스택은 pop 수행`

예시:
```
[1번 케이스]
냥..        # 변수1 = 1
냥냥...     # 변수2 = 2
냥~         # 스택에 1 push & last_result 갱신
냥냥~       # 스택에 2 push & last_result 갱신

[2번 케이스]
냥..        # 변수1 = 1
냥냥...     # 변수2 = 2
냥냥냥.     # 변수3 = 0
냥~         # 스택에 1 push & last_result 갱신
냥냥~       # 스택에 2 push & last_result 갱신
냐냐~       # 덧셈 연산, 스택 pop 2번 연산, last_result = 1 + 2, 스택에 push 
냥냥냥~     # 이전 명령어가 연산 명령어이므로, 변수3 = 3, 스택은 pop 수행
```

## 3. 연산(Operation)
연산은 스택 기반으로 수행된다. <br>
절차:
1. 변수 값들을 `<냥 N개>~` 명령어로 스택에 push
2. 연산 명령 실행 (최소 2개의 값이 스택에 있어야함)
3. 결과를 last_result에 갱신

### 3.1 연산 규칙
형식:
```
<냐 N개>~
```
|냐 개수|연산|
|---|---|
|2|덧셈|
|3|뺄셈|
|4|곱셈|
|5|나눗셈|
