# NyangLang 프로젝트 소개

**도전학기제 | 허민엽 | 2026년 4월**

---

## 1. 프로젝트 개요

### 1.1 한 줄 소개

> **NyangLang**은 한국어 고양이 울음소리(`냥`, `냐`, `야옹`)만으로 프로그램을 작성하는 에솔랭(Esoteric Programming Language)입니다.

### 1.2 에솔랭(Esolang)이란?

에솔랭은 실용성보다 독창성·실험성을 목표로 설계된 프로그래밍 언어입니다. 대표적인 예로 `Brainfuck`(8개 기호만 사용), `Whitespace`(공백 문자만 사용) 등이 있습니다.

에솔랭 제작은 언어 이론(어휘 분석, 구문 분석, 의미 분석)과 인터프리터 설계를 직접 구현해보는 학습 방법으로, **프로그래밍 언어론** 및 **컴파일러** 과목의 핵심 개념을 실전으로 체득할 수 있습니다.

## 2. 언어 설계 철학

### 2.1 핵심 기호 체계

| 기호 | 의미 | 역할 |
|------|------|------|
| `냥` | 변수 식별자 | `냥` 개수 = 변수 번호 |
| `냐` | 연산자 | `냐` 개수 = 연산 종류 |
| `야옹` | 문장 종결자 | 모든 커맨드 끝에 필수 |
| `.` / `,` | 정수 리터럴 | `.` = +1, `,` = -1 누적 합산 |
| `~` | 스택 push / 대입 | `~` = push, `~~` = 대입 |
| `!` / `!!` | 출력 | `!` = 10진수, `!!` = ASCII |
| `?` / `??` | 점프 / 입력 | infix = JUMP, suffix = INPUT |

### 2.2 설계 원칙

- **prefix ≠ suffix**: 동일 기호가 앞에 올 때와 뒤에 올 때 다른 의미를 가져 문법 모호성 최소화
- **야옹 종결자**: 모든 커맨드 끝에 `야옹`을 필수로 붙여 한 줄에 여러 커맨드를 명확하게 구분
- **스택 기반 연산**: 별도 레지스터 없이 스택과 변수 테이블만으로 모든 연산 처리
- **카운트 인코딩**: 기호의 반복 횟수 자체가 의미를 결정 (`냥` 1개 = 변수1, 2개 = 변수2)

## 3. 아키텍처

### 3.1 전체 파이프라인

```
     소스 코드 (.nyang 파일)
         │
         ▼
  ┌─────────────┐
  │    Lexer    │  lex_line(line) → List[Token] (토큰 리스트)
  │ 어휘 분석기 │  문자열을 의미 단위(토큰)로 분리
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │   Parser    │  parse_line(tokens) → List[Command] (커맨드 리스트)
  │ 구문 분석기 │  토큰 열을 실행 가능한 커맨드 구조체로 변환
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │ Interpreter │  run_program(lines) → 실행
  │ 인터프리터  │  PC(Program Counter) 기반 라인 순차 실행
  └─────────────┘
         │
         ▼
     출력 결과
```

### 3.2 파일 구조

```
src/nyang/
├── _00_types.py       # TokenType, Token, CommandKind, Command 타입 정의
├── _01_lexer.py       # 한 줄 → Token 리스트 (lex_line)
├── _02_parser.py      # Token 리스트 → Command 리스트 (parse_line, TokenStream)
├── _03_interpreter.py # PC 기반 실행 제어 (run_program) + CommandExecMixin 상속

web/                   # 웹 플레이그라운드
├── app.py             # FastAPI 서버, WebSocket 기반 실시간 실행
├── templates/         # Jinja2 HTML 템플릿 (VS Code 스타일 UI)
└── static/            # CSS, 이미지

examples/              # 예제 코드
├── 00_ex.nyang ~ 06_ex.nyang
```

## 4. 어휘 분석기 (Lexer) — `_01_lexer.py`

### 4.1 역할

소스 코드 한 줄을 입력받아 **의미 있는 최소 단위(Token)** 의 리스트로 변환합니다.
<br> 컴파일러 이론의 **어휘 분석(Lexical Analysis)** 단계에 해당합니다.

### 4.2 Token 구조

```
Token {
    type  : NYANG | NYA | YAONG | INT | TILDE | BANG | QUESTION
    value : 해당 기호의 연속 개수 또는 누적 정수값
}
```

| type | 대응 기호 | value 의미 |
|------|-----------|-----------|
| NYANG | `냥` | 연속 개수 (변수 번호) |
| NYA | `냐` | 연속 개수 (연산 종류) |
| YAONG | `야옹` | 없음 (종결자) |
| INT | `.` / `,` | `.`=+1, `,`=-1 누적 합산값 |
| TILDE | `~` | 연속 개수 (1=push, 2=대입) |
| BANG | `!` | 연속 개수 (1=10진수, 2=ASCII) |
| QUESTION | `?` | 연속 개수 (1=점프, 2=입력, 3=공백) |

### 4.3 동작 원리

`lex_line()` 함수는 문자를 한 글자씩 순회하는 **단일 패스(Single-Pass)** 방식입니다.

```
1. 주석 제거          : '#' 이후 문자 전부 무시
2. 야옹 종결자 감지   : '야' + '옹' 2글자 묶음을 YAONG 토큰으로 처리
3. 타입 전환          : 이전 문자와 다른 타입이 오면 이전 토큰을 저장하고 새 토큰 시작
4. 카운트 누적        : 같은 타입이 연속되면 count를 증가 (INT는 '.'=+1, ','=-1 누적)
```

### 4.4 토큰화 예시

입력: `냥냥...야옹...~야옹`

| 입력 문자 | 분류 | 처리 결과 |
|-----------|------|-----------|
| `냥` | NYANG | count=1 시작 |
| `냥` | NYANG | count=2 누적 |
| `.` | INT (타입 전환) | NYANG(2) 확정, count=1 시작 |
| `.` | INT | count=2 누적 |
| `.` | INT | count=3 누적 |
| `야옹` | YAONG (타입 전환) | INT(3) 확정, YAONG 추가 |
| `.` | INT (타입 전환) | count=1 시작 |
| `.` | INT | count=2 누적 |
| `.` | INT | count=3 누적 |
| `~` | TILDE (타입 전환) | INT(3) 확정, TILDE 추가 |
| `야옹` | YAONG (타입 전환) | TILDE(1) 확정,count=1 누적 |

**결과**: `[Token(NYANG, 2), Token(INT, 3), Token(YAONG, None), Token(INT, 3), Token(TILDE, 1), Token(YAONG, None)]`

→ 해석: 변수2를 3으로 초기화하는 선언문 후 정수 3을 스택에 PUSH <br>
→ 두 커맨드가 하나의 토큰 리스트에 순서대로 담김

## 5. 구문 분석기 (Parser) — `_02_parser.py`

### 5.1 역할

Lexer가 만든 Token 리스트를 입력받아 **실행 가능한 커맨드(Command)** 리스트로 변환합니다.
<br> 컴파일러 이론의 **구문 분석(Syntax Analysis)** 단계에 해당하며, `TokenStream` 커서로 토큰 리스트를 **단일 패스(Single-Pass)** 로 순회하면서 첫 번째 토큰만 보고 어떤 커맨드인지 즉시 결정하는 **예측적(Predictive)** 방식으로 구현되었습니다.

### 5.2 Command 구조

```
Command {
    kind        : VAR_DECL | VALUE_PUSH | VAR_PUSH | VAR_ASSIGN
                  OPERATION | INPUT | OUTPUT | JUMP | ...
    nyang_id    : 변수 번호  (해당 커맨드에서 사용하는 경우에만)
    int_value   : 정수 리터럴 값
    op_arity    : 연산 종류 (냐 개수: 2=덧셈 3=뺄셈 4=곱셈 5=나눗셈)
    output_form : 'decimal' | 'ascii'
    output_mode : 'newline' | 'inline' | 'space'
    jump_kind   : 점프 조건/목적지의 타입 조합
    condition   : 점프 조건값 (변수 번호 또는 리터럴)
    jump_line   : 점프 목적 라인 (변수 번호 또는 리터럴)
}
```

### 5.3 TokenStream — 커서(Cursor) 방식

Parser는 `TokenStream` 클래스로 토큰 리스트 위에 커서를 두고 순회합니다.

**핵심 메서드 두 가지:**

- **`peek()`** — 현재 커서 위치의 토큰을 반환하되 커서를 움직이지 않습니다. 
<br> 다음 토큰이 무엇인지 미리 확인해 어떤 커맨드를 파싱할지 결정하는 데 사용합니다.
- **`consume()`** — 현재 커서 위치의 토큰을 반환하고 커서를 한 칸 전진시킵니다. 해당 토큰을 실제로 소비(사용)할 때 호출합니다.

두 메서드의 조합으로 파서가 동작하는 방식:

```
peek()으로 다음 토큰 확인 → 어떤 커맨드인지 판단
consume()으로 해당 토큰 소비 → 커맨드 구조체에 값 저장
다시 peek() → 다음 토큰 확인 → ...
```

### 5.4 파싱 규칙 (Grammar)

첫 번째 토큰 타입을 보고 어떤 파서를 호출할지 결정합니다 (예측적 파싱).

```
첫 토큰이 NYANG → parse_nyang_command()
첫 토큰이 NYA   → parse_nya_command()
첫 토큰이 INT   → parse_int_command()
```

주요 파싱 패턴:

| 토큰 시퀀스 | CommandKind | 설명 |
|------------|-------------|------|
| `NYANG(N) INT(v) YAONG` | VAR_DECL | 변수N = v 선언 |
| `NYANG(N) TILDE(1) YAONG` | VAR_PUSH | 변수N을 스택에 push |
| `NYANG(N) TILDE(2) YAONG` | VAR_ASSIGN | 스택 top → 변수N 대입 |
| `INT(v) TILDE(1) YAONG` | VALUE_PUSH | 정수v를 스택에 push |
| `NYA(n) TILDE(1) YAONG` | OPERATION | 냐 n개 → 연산 수행 |
| `NYANG(N) QUESTION(2) YAONG` | INPUT | 변수N에 입력 |
| `NYANG(N) BANG(1) QUESTION(1) YAONG` | OUTPUT | 변수N 10진수 출력 (줄바꿈) |
| `NYANG(N) BANG(2) QUESTION(2) YAONG` | OUTPUT | 변수N ASCII 출력 (공백) |
| `NYANG(N) QUESTION(1) INT(L) YAONG` | JUMP | 변수N≠0 이면 L번 줄로 점프 |

### 5.5 야옹 종결자 검증

커맨드 하나를 파싱한 뒤, 반드시 `YAONG` 토큰이 있는지 검사합니다. 없으면 `SyntaxError`를 발생시킵니다.

### 5.6 파싱 예시

입력 토큰: `[NYANG(1), TILDE(1), YAONG, INT(3), TILDE(1), YAONG]`
```
1. peek → NYANG → parse_nyang_command() 호출
   consume NYANG(1)
   peek → TILDE, value=1 → VAR_PUSH 커맨드 생성 (nyang_id=1)
   consume TILDE
   peek → YAONG → 확인 후 소비
   → Command(VAR_PUSH, nyang_id=1) 추가

2. peek → INT → parse_int_command() 호출
   consume INT(3)
   peek → TILDE, value=1 → VALUE_PUSH 커맨드 생성 (int_value=3)
   consume TILDE
   peek → YAONG → 확인 후 소비
   → Command(VALUE_PUSH, int_value=3) 추가
```
결과: `[Command(VAR_PUSH, nyang_id=1), Command(VALUE_PUSH, int_value=3)]`

## 6. 인터프리터 (Interpreter) — `_03_interpreter.py`

### 6.1 역할

Parser가 만든 Command 리스트를 실제로 실행합니다.
**PC(Program Counter) 기반 실행 루프** 로 동작하며, 점프 명령으로 PC를 직접 제어해 조건 분기와 반복을 구현합니다.

### 6.2 런타임 상태

```
Interpreter {
    variables_table : {변수번호 → 정수값}   // 변수 테이블
    stack           : [정수, ...]           // 연산용 스택
    output_func     : 함수                  // 기본값 print, 웹에서는 교체
    input_func      : 함수                  // 기본값 input, 웹에서는 교체
    current_line    : 정수                  // 현재 실행 라인 (에러 보고용)
}
```

### 6.3 PC 기반 실행 루프

**주요 특징:**
- 한 줄에 여러 커맨드가 있어도 JUMP 커맨드가 나오면 즉시 해당 줄 실행을 중단
- 매 줄마다 Lex → Parse를 반복하는 **인터프리트 방식** (사전 컴파일 없음)
- `current_line`을 항상 갱신해 에러 발생 시 정확한 줄 번호 보고

### 6.4 Mixin 패턴 — 관심사 분리

커맨드별 실행 로직이 모두 `_03_interpreter.py` 한 파일에 있으면 코드가 방대해지므로, **CommandExecMixin** 클래스에 `_exec_*` 메서드를 모아두고 `Interpreter`가 이를 상속합니다.

```
CommandExecMixin          Interpreter
├── _exec_var_decl()    ←─ 상속 ─── run_program()
├── _exec_var_push()                execute_with_pc()
├── _exec_var_assign()              variables_table
├── _exec_value_push()              stack
├── _exec_operation()               output_func
├── _exec_input()                   input_func
├── _exec_output()
└── _exec_jump()
```

→ `interpreter.py`는 **제어 흐름**에만 집중하고, 실행 로직은 Mixin에 위임합니다.

### 6.5 각 커맨드 실행 로직

**VAR_DECL — 변수 선언**
```
variables_table[cmd.nyang_id] ← cmd.int_value
// 예: 냥냥...야옹 → variables_table[2] = 3
```

**VAR_PUSH / VALUE_PUSH — 스택에 push**
```
VAR_PUSH  : stack.push( variables_table[cmd.nyang_id] )
VALUE_PUSH: stack.push( cmd.int_value )
```

**OPERATION — 사칙연산**
```
b ← stack.pop()          // 나중에 push된 값
a ← stack.pop()          // 먼저 push된 값
result ← a op b          // op_arity: 2=덧셈 3=뺄셈 4=곱셈 5=나눗셈
stack.push(result)
```

**JUMP — 조건 점프 (4가지 변형)**

| jump_kind | 조건 | 목적지 |
|-----------|------|--------|
| `int?int` | 정수 리터럴 | 정수 리터럴 라인 |
| `int?nyang` | 정수 리터럴 | 변수값 라인 |
| `nyang?int` | 변수값 | 정수 리터럴 라인 |
| `nyang?nyang` | 변수값 | 변수값 라인 |

```
condition ← 리터럴 또는 variables_table에서 조건값 결정
target    ← 리터럴 또는 variables_table에서 목적 라인 결정

if condition ≠ 0: return target - 1   // 1-indexed → 0-indexed PC
else:             return pc + 1       // 조건 거짓 → 다음 줄
```

**OUTPUT — 출력 (output_func 주입)**
```
value ← variables_table[cmd.nyang_id]
out   ← chr(value)  if output_form == 'ascii'
        value       if output_form == 'decimal'
end   ← '\n' | '' | ' '   (output_mode에 따라)
output_func(out, end)
```

`output_func`를 `print` 대신 문자열 수집 함수로 교체하면, 웹 백엔드에서 출력 결과를 WebSocket으로 스트리밍할 수 있습니다 (의존성 주입 패턴).

## 7. 전체 실행 추적 예시

### 코드: 두 변수를 더해 출력

```
냥...야옹          # 1번 줄
냥냥..야옹         # 2번 줄
냥~야옹 냥냥~야옹 냐냐~야옹 냥냥냥~~야옹   # 3번 줄
냥냥냥!?야옹       # 4번 줄
```

### 실행 추적

**1번 줄** `냥...야옹`
- Lex → `[NYANG(1), INT(3), YAONG]`
- Parse → `[Command(VAR_DECL, nyang_id=1, int_value=3)]`
- Exec → `variables_table = {1: 3}`

**2번 줄** `냥냥..야옹`
- Lex → `[NYANG(2), INT(2), YAONG]`
- Parse → `[Command(VAR_DECL, nyang_id=2, int_value=2)]`
- Exec → `variables_table = {1: 3, 2: 2}`

**3번 줄** `냥~야옹 냥냥~야옹 냐냐~야옹 냥냥냥~~야옹`
- Lex → `[NYANG(1),TILDE(1),YAONG, NYANG(2),TILDE(1),YAONG, NYA(2),TILDE(1),YAONG, NYANG(3),TILDE(2),YAONG]`
- Parse → 커맨드 4개:
  1. `VAR_PUSH(nyang_id=1)` → stack: [3]
  2. `VAR_PUSH(nyang_id=2)` → stack: [3, 2]
  3. `OPERATION(op_arity=2)` → pop 2,3 → 3+2=5 → stack: [5]
  4. `VAR_ASSIGN(nyang_id=3)` → stack pop → variables_table[3] = 5

**4번 줄** `냥냥냥!?야옹`
- Lex → `[NYANG(3), BANG(1), QUESTION(1), YAONG]`
- Parse → `Command(OUTPUT, nyang_id=3, output_form='decimal', output_mode='newline')`
- Exec → `print(5)` → **출력: 5**

## 8. 웹 플레이그라운드

NyangLang 코드를 브라우저에서 바로 실행해볼 수 있는 웹 인터페이스를 제공합니다.

### 8.1 기술 스택

- **백엔드**: FastAPI + WebSocket (실시간 입출력 스트리밍)
- **프론트엔드**: HTML/CSS/JS (VS Code 스타일 UI)
- **실행**: `uvicorn web.app:app --reload`

### 8.2 WebSocket 기반 실시간 실행

```
브라우저 ──[코드 전송]──► FastAPI WebSocket
                              │
                    별도 스레드에서 Interpreter 실행
                              │
          출력 발생 시 ◄──[output 메시지]── Interpreter
          입력 필요 시 ──[input_request]──► 브라우저 입력창
          완료 시     ◄──[done 메시지]────
```

기존 HTTP 방식은 실행이 끝날 때까지 응답을 기다려야 하지만, WebSocket을 사용해 출력을 **실시간 스트리밍** 하고 **입력 대화**도 지원합니다.

### 8.3 Output 주입 패턴

```python
# 일반 실행 (CLI)
interp = Interpreter(output_func=print, input_func=input)

# 웹 실행 (WebSocket)
interp = Interpreter(
    output_func=lambda msg, end="": ws.send({"type":"output","data":msg}),
    input_func=lambda prompt: websocket_input_wait(prompt)
)
```

인터프리터 코드를 전혀 수정하지 않고 `output_func`와 `input_func`만 교체하여 CLI와 웹 양쪽을 지원합니다.

## 9. 문법 명세 요약

### 9.1 변수와 연산

```
냥??야옹           # 변수1에 입력받기 (INPUT)
냥...야옹          # 변수1 = 3 으로 초기화 (VAR_DECL)
냥~야옹            # 변수1 값을 스택에 push (VAR_PUSH)
냥~~야옹           # 스택 top → 변수1에 대입 (VAR_ASSIGN)
...~야옹           # 정수 3을 스택에 push (VALUE_PUSH)
냐냐~야옹          # 스택에서 2개 pop → 덧셈 → 결과 push (OPERATION)
```

**연산 종류** (`냐` 개수로 구분):

| 냐 개수 | 연산 |
|---------|------|
| 2 | 덧셈 (a + b) |
| 3 | 뺄셈 (a - b) |
| 4 | 곱셈 (a × b) |
| 5 | 나눗셈 (a ÷ b, 정수) |

### 9.2 입출력

```
냥N??야옹          # 변수N에 정수 입력
냥N!?야옹          # 변수N을 10진수로 출력 (줄바꿈)
냥N!!??야옹        # 변수N을 ASCII 문자로 출력 (공백 구분)
냐!야옹            # 현재 스택 상태 출력 (디버그)
냐!!야옹           # 변수 테이블 출력 (디버그)
```

### 9.3 점프문 (조건 분기)

```
냥N?...야옹        # 변수N ≠ 0 이면 3번째 줄로 점프
냥N?냥M야옹        # 변수N ≠ 0 이면 변수M번째 줄로 점프
...?....야옹       # 정수3 ≠ 0 이면 4번째 줄로 점프
```

점프 조건이 **0** 이면 다음 줄로 진행합니다.

### 10.1 Hello World (일부)

### 10.2 구구단 (2~9단)

```
냥..야옹                    # i = 2 (단)
냥냥.야옹                   # j = 1 ← 내부 루프 시작
냥~야옹 냥냥~야옹 냐냐냐냐~야옹 냥냥냥~~야옹    # 변수3 = i × j
냥냥냥!???야옹              # 결과 출력 (공백 구분)
냥냥~야옹 .~야옹 냐냐~야옹 냥냥~~야옹           # j = j + 1
냥냥~야옹 ..........~야옹 냐냐냐~야옹 냥냥냥냥~~야옹  # 변수4 = j - 10
냥냥냥냥?...야옹            # 변수4 ≠ 0 이면 3번 줄로 (j < 10)
..........!!??야옹          # 줄바꿈 출력
냥~야옹 .~야옹 냐냐~야옹 냥~~야옹               # i = i + 1
냥~야옹 ..........~야옹 냐냐냐~야옹 냥냥냥냥~~야옹  # 변수4 = i - 10
냥냥냥냥?..야옹             # 변수4 ≠ 0 이면 2번 줄로 (i < 10)
```

### 10.3 입력받은 단의 구구단

```
냥??야옹                    # 변수1(n) = 입력
냥냥.야옹                   # 변수2(j) = 1
...  (n * j 계산 및 출력, j 증가, 루프 조건 체크)
냥냥냥냥냥냥?....야옹        # j ≠ 6이면 루프 계속
```
