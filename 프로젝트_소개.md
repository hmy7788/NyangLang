**도전학기제 | 허민엽 | 2026년 4월**

---

## 1. 프로젝트 개요

### 1.1 한 줄 소개

> **NyangLang**은 한국어 고양이 울음소리(`냥`, `냐`, `야옹`)만으로 프로그램을 작성하는 에솔랭(Esoteric Programming Language)입니다.

### 1.2 에솔랭(Esolang)이란?

에솔랭은 실용성보다 독창성·실험성을 목표로 설계된 프로그래밍 언어입니다. 대표적인 예로 `Brainfuck`(8개 기호만 사용), `Whitespace`(공백 문자만 사용) 등이 있습니다. 
<br> 에솔랭 제작은 언어 이론(어휘 분석, 구문 분석, 의미 분석)과 인터프리터 설계를 직접 구현해보는 학습 방법으로, 프로그래밍 언어론 및 컴파일러 과목의 핵심 개념을 실전으로 체득할 수 있습니다.

## 2. 언어 설계 철학

### 2.1 핵심 기호 체계

| 기호 | 의미 | 역할 |
|------|------|------|
| `냥` | 변수 식별자 | `냥` 개수 = 변수 번호 |
| `냐` | 연산자 | `냐` 개수 = 연산 종류 |
| `야옹` | 문장 종결자 | 모든 커맨드 끝에 필수 |
| `.` / `,` | 정수 리터럴 | `.` = +1, `,` = -1 |
| `~` | 스택 push / 대입 | `~` = push, `~~` = 대입 |
| `!` / `!!` | 출력 | `!` = 10진수, `!!` = ASCII |
| `?` / `??` | 점프 / 입력 | infix = JUMP, suffix = INPUT |

### 2.2 설계 원칙

- **prefix ≠ suffix**: 동일 기호가 앞에 올 때와 뒤에 올 때 다른 의미를 가져 문법 모호성을 최소화
- **야옹 종결자**: 모든 커맨드 끝에 `야옹`을 필수로 붙여 한 줄에 여러 커맨드를 명확하게 구분
- **스택 기반 연산**: 별도 레지스터 없이 스택과 변수 테이블만으로 모든 연산 처리

## 3. 아키텍처

### 3.1 전체 파이프라인

```
소스 코드 (.nyang)
       ↓
   [ Lexer ]         문자열 → Token 리스트
       ↓
   [ Parser ]        Token 리스트 → Command 리스트
       ↓
 [ Interpreter ]     Command 리스트 실행 (PC 기반)
       ↓
    출력 결과
```

### 3.2 파일 구조

```
src/nyang/             # lexer, parser, interpreter 코드 
├── _00_types.py       # TokenType, Token, CommandKind, Command 타입 정의
├── _01_lexer.py       # 코드 한 줄 → Token 리스트 (lex_line)
├── _02_parser.py      # Token 리스트 → Command 리스트 (TokenStream, parse_line)
├── _03_interpreter.py # PC 기반 실행 제어 (run_program)
├── _04_cli.py         # argparse CLI (nyang 명령어)
└── _05_cmd.py         # CommandExecMixin (커맨드별 실행 메서드)

web/                   # 웹 플레이그라운드
├── app.py             # FastAPI 웹 플레이그라운드 서버
├── templates/         # Jinja2 HTML 템플릿
└── static/            # CSS, JS

examples/              # 예제 코드
├── 00_ex.nyang        # 덧셈 (변수 2개 입력 후 합 출력)
├── 01_ex.nyang        # 산술 연산 + ASCII 출력
├── 02_ex.nyang        # 스택/변수 디버그 출력
├── 03_ex.nyang        # Hello World!
└── 04_ex.nyang        # 짝수/홀수 판별
```

### 3.3 주요 설계 결정

**Mixin 패턴**: 커맨드별 실행 메서드(`_exec_*`)를 `CommandExecMixin`으로 분리하여 `interpreter.py`가 제어 흐름에만 집중하도록 구성

**Output 주입**: `Interpreter`의 `output_func`를 기본값 `print`에서 문자열 수집 함수로 교체 가능하게 설계 → 웹 API에서 출력 결과를 JSON으로 반환할 때 활용

**Token Splitting**: `TokenStream.consume_value()`로 하나의 토큰을 가상으로 쪼개어 사용 가능 — 예: `BANG(3)` 토큰에서 1만 소비하면 `BANG(2)`가 남음

## 4. 문법 명세 요약

### 4.1 변수와 연산

```
냥??야옹           # 변수1에 입력받기 (INPUT)
냥...야옹          # 변수1 = 3 으로 초기화 (VAR_DECL)
냥~야옹            # 변수1 값을 스택에 push (VAR_PUSH)
냥~~야옹           # 스택 top → 변수1에 대입 (VAR_ASSIGN)
...~야옹           # 정수 3을 스택에 push (VALUE_PUSH)
냐냐~야옹          # 스택에서 2개 pop → 덧셈 → 결과 push (OPERATION)
```

**연산 종류** (`냐` 개수로 구분):

| 냐 개수 | 연산 |
|---------|------|
| 2 | 덧셈 |
| 3 | 뺄셈 |
| 4 | 곱셈 |
| 5 | 나눗셈 |

### 4.2 입출력

```
냥N??야옹          # 변수N에 정수 입력
냥N!?야옹          # 변수N을 10진수로 출력 (줄바꿈)
냥N!!??야옹        # 변수N을 ASCII 문자로 출력 (공백 구분)
냐!야옹            # 현재 스택 상태 출력 (디버그)
냐!!야옹           # 변수 테이블 출력 (디버그)
```

### 4.3 점프문 (조건 분기)

```
냥N?...야옹        # 변수N ≠ 0 이면 3번째 줄로 점프
냥N?냥M야옹        # 변수N ≠ 0 이면 변수M번째 줄로 점프
...?....야옹       # 스택 top = 3 이면 4번째 줄로 점프
```

점프 조건이 **거짓(0)** 이면 다음 줄로 진행합니다.

## 5. 예제 코드

### 5.1 Hello World

```
..........~야옹 .......~야옹
냐냐냐냐~야옹 냥~~야옹 냥~야옹 ..~야옹 냐냐~야옹 냥~~야옹
냥!!??야옹   # → 'H' (72) 출력
```

10과 7을 스택에 push → 곱셈(70) → 2를 더해 72(H)를 얻어 ASCII 출력

### 5.2 짝수/홀수 판별

```
냥??야옹                    # 변수1 ← 입력
냥?.......야옹              # 변수1 ≠ 0 → 7번 줄로 (홀수 경로)
.!?야옹                     # 1 출력 (짝수)
.?...................야옹   # 19번 줄로 (종료)
냥~야옹 .~야옹              # 변수1, 1 을 스택에 push
냐냐냐~야옹                 # 뺄셈: 변수1 - 1
냥~~야옹                    # 변수1 = 결과
...                        # (반복)
```

1씩 감소하며 짝수/홀수 위치를 번갈아 체크하는 방식으로 판별합니다.

## 6. 웹 플레이그라운드

NyangLang 코드를 브라우저에서 바로 실행해볼 수 있는 웹 인터페이스를 제공합니다.

- **기술 스택**: FastAPI + Jinja2 + HTML/CSS/JS
- **동작 방식**: 사용자가 코드를 입력 → `POST /run` API 호출 → 인터프리터 실행 → 결과 JSON 반환
- **실행 방법**: `uvicorn web.app:app --reload`
